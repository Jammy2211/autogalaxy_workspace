# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

The **autogalaxy_workspace** is a collection of Python scripts and Jupyter notebooks for galaxy morphology modeling using **PyAutoGalaxy**. It is not a library — it is a workspace of runnable examples. The core library lives in a separate `autogalaxy` package (installed via pip).

Dependencies: `autogalaxy`, `numba`, `autofit` (the probabilistic programming framework used for model composition and non-linear searches).

Python version: 3.11

## Running Scripts

Scripts are run from the repository root so that relative paths to `dataset/` and `output/` resolve correctly:

```bash
python scripts/imaging/modeling.py
python scripts/imaging/simulator.py
python scripts/interferometer/modeling.py
```

Jupyter notebooks mirror the scripts in `notebooks/` and use `# %%` markers to separate cells.

**Integration testing / fast mode**: Set `PYAUTOFIT_TEST_MODE=1` to run any example script without performing full non-linear search sampling. The search is skipped and a mock result is returned immediately, making this ideal for high-level integration testing of scripts end-to-end:

```bash
PYAUTOFIT_TEST_MODE=1 python scripts/imaging/modeling.py
```

**Fast smoke tests**: For maximum speed, combine test mode with small datasets and fast plots:

```bash
PYAUTOFIT_TEST_MODE=2 PYAUTO_WORKSPACE_SMALL_DATASETS=1 PYAUTO_FAST_PLOTS=1 python scripts/imaging/modeling.py
```

- `PYAUTO_WORKSPACE_SMALL_DATASETS=1` — caps all grids/masks to 15x15 pixels at 0.6"/px, making simulators and all downstream computations dramatically faster. Delete `dataset/` when toggling this variable.
- `PYAUTO_FAST_PLOTS=1` — skips `plt.tight_layout()` and critical curve/caustic overlay computation in subplot functions.

**Codex / sandboxed runs**: when running from Codex or any restricted environment, set writable cache directories so `numba` and `matplotlib` do not fail on unwritable home or source-tree paths:

```bash
NUMBA_CACHE_DIR=/tmp/numba_cache MPLCONFIGDIR=/tmp/matplotlib python scripts/imaging/modeling.py
```

This workspace is often imported from `/mnt/c/...` and Codex may not be able to write to module `__pycache__` directories or `/home/jammy/.cache`, which can cause import-time `numba` caching failures without this override.

**Parallelization bug fix**: On Linux/Windows, model fits using JAX + multiprocessing may error unless the fit is wrapped in `if __name__ == "__main__": fit()`. See `scripts/guides/modeling/bug_fix.py` for the pattern.

## Testing All Scripts

Run all scripts end-to-end using the provided runner:

```bash
bash run_all_scripts.sh
```

- Sets `PYAUTOFIT_TEST_MODE=1` automatically (skips non-linear sampling)
- Failure logs written to `failed/<relative_path>.log`; execution continues after each failure
- Final summary shows pass/fail/skipped counts

To run a single script in test mode:

```bash
PYAUTOFIT_TEST_MODE=1 python3 scripts/imaging/modeling.py
```

### Script ordering rules

1. **`start_here.py` runs first**: if a folder contains `start_here.py`, it is executed before any other scripts in that folder and before descending into subfolders. Some scripts depend on results generated by `start_here.py`.

2. **Skip list from `no_run.yaml`**: `../PyAutoBuild/autobuild/config/no_run.yaml` lists scripts that must not be run, under the `autogalaxy` key. The runner reads this file, prints all skipped scripts with their reasons, and flags any marked `FUTURE_PR` (those with a bug report or issue that should be fixed in a future PR).

   To add a new skip entry, add a line to `no_run.yaml` under `autogalaxy:`:
   ```yaml
   autogalaxy:
   - script_name  # Reason it cannot run yet
   - path/to/script  # bug https://github.com/...
   ```
   Scripts with a bug link or the words "bug"/"issue"/"fix"/"update"/"pr" in the comment are flagged as `[TODO - should run after a future PR]`.

   Matching is by basename stem or relative path suffix (e.g. `searches` matches any script under a `searches/` folder).

## Workspace Structure

```
scripts/         Python script examples (canonical source — notebooks are generated from these)
notebooks/       Jupyter notebook versions of scripts
dataset/         Input .fits files and other data (imaging, interferometer, multi-band)
config/          YAML configuration files for PyAutoGalaxy behaviour
output/          Model-fit results written here at runtime
```

Sub-packages within `scripts/` and `notebooks/`:
- `imaging/` — CCD imaging (HST, JWST, Euclid) modeling
- `interferometer/` — ALMA/JVLA uv-plane modeling
- `multi/` — Multi-wavelength simultaneous modeling
- `guides/` — API guides: modeling, results, plotting, over-sampling, HPC
- `howtogalaxy/` — Tutorial lecture series
- `ellipse/` — Ellipse fitting (non-parametric morphology)

Each data-type package contains: `modeling.py`, `simulator.py`, `fit.py`, `features/`, `data_preparation/`, `likelihood_function.py`.

## Core API Patterns

**Imports** (standard across all scripts):
```python
import autofit as af
import autogalaxy as ag
import autogalaxy.plot as aplt
```

**Standard modeling workflow**:
1. Load dataset from `.fits` files via `ag.Imaging.from_fits(...)` or `ag.Interferometer.from_fits(...)`
2. Apply a 2D mask: `ag.Mask2D.circular(...)`
3. Apply adaptive over-sampling: `ag.util.over_sample.over_sample_size_via_radial_bins_from(...)`
4. Compose a model using `af.Model` / `af.Collection` with `ag.Galaxy` and light profiles from `ag.lp_linear.*`
5. Configure a non-linear search (default: `af.Nautilus`)
6. Create an `ag.AnalysisImaging(dataset=dataset, use_jax=True)` object
7. Run `search.fit(model=model, analysis=analysis)`

**Linear light profiles** (`ag.lp_linear.*`) are preferred over standard profiles (`ag.lp.*`) because they solve for `intensity` via linear inversion rather than as a free parameter, reducing model dimensionality.

**JAX acceleration**: Pass `use_jax=True` to all `Analysis` objects. The `n_batch` parameter on the search controls GPU batch size (affects VRAM). Use `analysis.print_vram_use(model=model, batch_size=search.batch_size)` to estimate VRAM before running.

## Configuration

All config lives in `config/`:
- `general.yaml` — Global settings (HPC mode, numba, inversion checks, output format)
- `non_linear/nest.yaml`, `mcmc.yaml`, `mle.yaml` — Search-specific defaults
- `priors/` — Default priors per model component
- `visualize/` — Controls what images are auto-generated during fits
- `notation.yaml` — Parameter short-hand labels used in plots

To enable HPC mode (disables GUI, reduces logging): set `hpc_mode: true` in `config/general.yaml`.

## Notebooks vs Scripts

Notebooks in `notebooks/` are generated from the Python files in `scripts/`. **Always edit the `.py` scripts**, not the notebooks directly. The `# %%` marker alternates between code and markdown cells.

### Building Notebooks

Notebooks are generated from Python scripts using `generate.py` from the `PyAutoBuild` repo. Run from the workspace root:

```bash
PYTHONPATH=../PyAutoBuild/autobuild python3 ../PyAutoBuild/autobuild/generate.py autogalaxy
```

This converts every `.py` file in `scripts/` to a `.ipynb` in `notebooks/` by:
1. Converting triple-quoted docstrings into `# %%` Jupyter cell markers
2. Running `ipynb-py-convert` to produce `.ipynb` files
3. Restoring commented Jupyter magic commands (e.g. `# %matplotlib` → `%matplotlib`)

Use the `/generate_and_merge` skill to build notebooks, commit, push, raise a PR, and merge into `main`.

## Output

Results are written to `output/` under a path determined by `path_prefix`, `name`, and `unique_tag` passed to the search. The unique identifier is hash-based on the model + search + dataset, so re-running the same configuration resumes from existing results automatically.

## Line Endings — Always Unix (LF)

All files in this project **must use Unix line endings (LF, `\n`)**. Windows/DOS line endings (CRLF, `\r\n`) will break shell scripts and Python files on HPC systems.

**When writing or editing any file**, always produce Unix line endings. Never write `\r\n` line endings.

After creating or copying files, verify and convert if needed:

```bash
# Check for DOS line endings
file scripts/imaging/modeling.py   # should say "ASCII text", not "CRLF"

# Convert a single file
dos2unix scripts/imaging/modeling.py

# Convert all shell scripts and Python files in the project
find . -type f \( -name "*.py" -o -name "*.sh" \) | xargs dos2unix
```

Prefer simple shell commands.
Avoid chaining with && or pipes.
